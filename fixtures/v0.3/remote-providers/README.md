# Remote provider conformance fixtures

These project-authored fixtures exercise only `remote-https-spki-v1`. The source and deterministic request/response replay are generated by `providers/reference-remote/worker.py --generate-fixtures fixtures/v0.3/remote-providers`.

The live test creates an ephemeral self-signed loopback TLS certificate at test time and binds its SPKI digest explicitly. No private key, credential, third-party source or service response is committed. Replay proves mapping only; live loopback tests prove the client transport behavior.
