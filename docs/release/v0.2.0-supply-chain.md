# AECCTX 0.2.0 Supply-chain, Provider, Security and Privacy Report

## Distribution boundary

- Core dependency: `jsonschema[format]>=4.23,<5`.
- Optional extras are pinned or bounded in `pyproject.toml`; clean core operation requires no adapter, provider, Codex, network or LLM.
- Wheel and sdist contain the Apache-2.0 core, public schemas and conformance material. The plugin is a separate deterministic ZIP.
- `SHA256SUMS` covers wheel, sdist and plugin archive. The SPDX 2.3 SBOM enumerates core and optional dependency packages from the lockfile.

## Restricted providers

Tesseract OCR, OCP/OCCT STEP/IGES and GNU LibreDWG execute only through the exact reviewed external-provider profiles. Their native/GPL runtimes and images are not redistributed in the wheel, sdist or plugin. Live platform claims remain narrower than portable offline replay claims. RVT has no accepted provider.

## Security and privacy

Inputs remain untrusted data. Release gates scan artifacts for RVT/provider binaries, credentials and WoodFraming/consumer dependencies; provider descriptors retain no-network/default-deny and bounded-resource requirements. OCR/provider invocation is explicit opt-in. Signing uses caller-supplied keys and offline policy; it never discovers trust roots or equates validity with approval.

## Licenses and residuals

Reviewed dependency/provider notes live under `docs/licenses/` and `docs/security/`. Dependency licenses remain the responsibility of the installing operator for selected extras. No proprietary binary is included. Unsupported and blocked profiles remain visible in the capability matrix and claim registry.
