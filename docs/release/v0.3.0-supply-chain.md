# AECCTX 0.3.0 Supply-chain, Provider, Security and Privacy Report

## Distribution boundary

- The core dependency remains `jsonschema[format]>=4.23,<5`; optional extras remain explicit in `pyproject.toml` and `uv.lock`.
- The wheel contains the Apache-2.0 Python implementation and packaged schemas. The sdist adds public tests, fixtures, provider source recipes, governance and conformance evidence. The inspector is a separate deterministic ZIP.
- `SHA256SUMS` covers wheel, sdist and inspector ZIP. The deterministic SPDX 2.3 SBOM enumerates the project and locked direct/optional dependency set.
- Clean core and all-extras installations run in separate Python 3.12 environments during release verification.

## Restricted providers and licenses

Tesseract, OCP/OCCT and GNU LibreDWG execute only through their exact reviewed external profiles. Their native runtimes, OCI images and restricted/GPL binaries are not shipped in the wheel, sdist or inspector ZIP. Provider source recipes and receipts are evidence, not bundled executables. RVT has no accepted provider, runtime, fixture corpus or semantic adapter.

Project-authored fixtures are Apache-2.0 unless their origin file says otherwise. Unchanged buildingSMART IDS fixtures retain their recorded CC BY-ND 4.0 provenance. Optional dependencies and external runtimes retain the license notes under `docs/licenses/`; redistribution remains the distributor's responsibility.

## Security and privacy

Release verification rejects native executable payloads, RVT v0.3 payloads/adapters and consumer-specific code in distributed artifacts. It also runs the existing path, archive, process, network, resource, malformed-output, prompt-injection, trust and policy adversarial suites.

Core operations remain offline and LLM-independent. Remote provider invocation, source upload, credentials, retention, telemetry and jurisdiction are explicit caller policy. No credential, production key, proprietary source or WoodFraming dependency is included.

## Residual risk

The exact unsupported and partial ceilings remain in `docs/capability-matrix.md`. In particular, native restricted-provider execution, RVT, replay-only availability, unlisted providers/platforms, external trust discovery, marketplace behavior, hidden source geometry, survey authority and consumer approval remain non-claims.
