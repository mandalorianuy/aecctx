# AECCTX 0.3.0

AECCTX 0.3.0 closes the governed post-v0.2 functional-debt line while preserving v0.1/v0.2 package compatibility. Install the reviewed artifact directly:

```bash
python -m pip install https://github.com/mandalorianuy/aecctx/releases/download/v0.3.0/aecctx-0.3.0-py3-none-any.whl
```

The release adds exact Linux arm64/amd64 OCI evidence, a closed optional remote-provider protocol, expanded IFC/DXF/OCR/vision/mesh/STEP/IGES/DWG profiles, advanced optional offline trust, expanded bounded IDS, and a reproducible inspector distribution. The aggregate registry maps 19 public outcomes: 17 bounded positive profiles and two explicit `unsupported` outcomes.

RVT remains documented blocked and public `unsupported`. Native restricted-provider sandboxes remain executable `unsupported`. Replay-only availability, unlisted formats/providers/platforms, hidden source geometry, survey authority, marketplace behavior, production trust/key custody, proprietary binaries, consumer mappings, engineering approval and WoodFraming integration are not release claims.

The release assets are the wheel, source distribution, inspector ZIP, `SHA256SUMS` and SPDX 2.3 SBOM. Verify all downloaded assets with:

```bash
sha256sum -c SHA256SUMS
```

See `docs/compatibility-v0.3.md`, `docs/release/v0.3.0-evidence-index.md` and `docs/release/v0.3.0-supply-chain.md` for exact boundaries.
